mochaにおけるdescribeのベストプラクティス

mochaのベストプラクティス

Node.jsのテスティングフレームワークであるmochaで困ったことがあったので解決策の覚書

起きたこと

  • テスト用にデータベースを作って、開発メンバーで共有して使っている
  • そのデータベースをいじって色々テストする
    • もちろん他のテストに影響を与えないように初期化しながら使う
  • あるとき、何もテストが走っていない状態でそのデータベースが初期化されていなかった

原因

  • テストが終わったあとにデータベースをinitializeできていなかった
  • beforeEachを使っていたが、afterはなかった

解決策

  • beforeEachだけだとitの直前のタイミングでしかinitializeが走らない
  • テストが終わっても最後のitのあとにデータベースにレコードが残っちゃう

  • afterも組み合わせてdescribeが終わったら綺麗にしてから帰りましょう、というお話

  • 往々にして、テストケースの最後は異常系とかにしてしまう

    • データベースに更新がかからないテストケースなので、afterしなくても平気な場合が大半
    • でも明示的にinitializeして気持ちよくテストできるようにしようね

beforEachのみで
テストケースの最後に異常系を置いたつもりが正常にデータベースが更新されていてテストが落ちた
みたいなことが起きると事件になります
気をつけましょう(経験談)

VSCode_win10からコミットする

win10からやるならコマンドはgitしか使いたくない人向け。

VSCode_win10でGitHubにえいってする

Git Bash

  1. $ git config --global user.name 'ユーザー名'
  2. $ git config --global user.email 'メールアドレス@gmail.com'

Git Gui

  1. Help > Show SSH Key
  2. Generate Key

ブラウザ(GitHub)

  1. SSH Keyの登録

VSCode

  1. $ git init
  2. $ git remote add origin https://github.com/***/***.git
  3. コマンドラインorソース管理メニューでadd commit push

以上。
gitだけはターミナルでやった方が確実。
VSCodeのソース管理タブはいまいち・・・。
